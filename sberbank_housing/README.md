# Sberbank Russian Housing Market

Решение задачи [Kaggle: Sberbank Russian Housing Market](https://www.kaggle.com/competitions/sberbank-russian-housing-market) — предсказание цен на жильё на российском рынке по данным Сбербанка.

---

## О задаче

**Тип:** регрессия (regression)  
**Цель:** предсказать целевую переменную `price_doc` — цену сделки купли-продажи недвижимости в рублях.

Данные охватывают сделки с недвижимостью (в основном Москва и область) за период 2011–2016 годов. Помимо характеристик самих объектов, в наборе есть признаки района (района/района), инфраструктуры и макроэкономические показатели по дате сделки.

---

## Данные

Структура в репозитории:

```
data/raw/
├── train.csv          # обучающая выборка (id, признаки, price_doc)
├── test.csv           # тестовая выборка (id, признаки)
├── macro.csv          # макроэкономические показатели по датам
├── sample_submission.csv  # шаблон сабмита (id, price_doc)
└── data_dictionary.txt    # описание признаков
```

### Основные признаки (train.csv / test.csv)

| Признак | Описание |
|--------|----------|
| `price_doc` | Цена сделки (целевая переменная, только в train) |
| `id` | Идентификатор сделки |
| `timestamp` | Дата сделки |
| `full_sq` | Общая площадь, м² (включая лоджии, балконы) |
| `life_sq` | Жилая площадь, м² |
| `floor`, `max_floor` | Этаж квартиры и этажность дома |
| `material` | Материал стен |
| `build_year` | Год постройки |
| `num_room` | Количество комнат |
| `kitch_sq` | Площадь кухни |
| `state` | Состояние квартиры |
| `product_type` | Тип: для себя (OwnerOccupier) или инвестиция (Investment) |
| `sub_area` | Район (Raion) |

Дополнительно в данных есть десятки признаков по району и окружению:

- население и демография района (`full_all`, `male_f`, `female_f`, возрастные группы);
- инфраструктура: метро (`metro_min_walk`, `metro_km_walk`), школы, парки, ТЦ, офисы, кафе в радиусах 500/1000/…/5000 м;
- расстояния до МКАД, ТТК, Садового кольца, Кремля и т.д.;
- экология, наличие промзон, ТЭЦ и т.п.

### macro.csv

Один ряд на дату: нефть (Urals, Brent), курс USD/EUR к рублю, ВВП, инфляция (CPI, PPI), ипотека (объём, ставка), доходы, зарплаты, строительство, демография и др. Признаки нужно объединять с `train`/`test` по полю `timestamp`.

## Метрика

На платформе Kaggle для этой задачи использовалась метрика **RMSE** (Root Mean Square Error) по логарифму цены или по самой цене — уточняется в описании соревнования. Цель — минимизировать ошибку предсказания цен.

---

## Подходы к решению

### 1. Разведка и предобработка (EDA)

- Анализ распределения `price_doc` (часто используют `log1p(price_doc)` для симметрии и устойчивости к выбросам).
- Поиск выбросов по площади (`full_sq`, `life_sq`), этажу, году постройки, числу комнат; решение — обрезка или отдельная обработка.
- Анализ пропусков: много признаков с NA (материал, год постройки, этажность и др.) — импутация (медиана, модель, категория «unknown») или бинарные флаги «признак отсутствует».
- Анализ по времени: тренды цен, сезонность; объединение с `macro.csv` по `timestamp`.

### 2. Feature engineering

- **Макро:** присоединение к сделкам по `timestamp` показателей из `macro.csv` (нефть, курс, ипотека, доходы и т.д.); при необходимости лаги (например, сдвиг на 1–3 месяца).
- **Площади и пропорции:** `life_sq/full_sq`, `kitch_sq/full_sq`, цена за м² (на трейне можно считать от агрегатов по району/времени).
- **Локация:** признаки района (`sub_area`), расстояния до центра/метро/МКАД; часто полезны агрегаты по району (медианная цена, средняя площадь и т.д.).
- **Время:** год/месяц/квартал из `timestamp`, число дней от стартовой даты.
- **Категории:** кодирование `material`, `state`, `product_type`, `sub_area` (One-Hot, Target Encoding или аналоги).
- Сокращение мультиколлинеарности: отбор признаков, PCA/другие методы уменьшения размерности при необходимости.

### 3. Модели

- **Линейные:** Ridge/Lasso регрессия (часто по логарифму цены) — быстрый бейзлайн и отбор признаков.
- **Градиентный бустинг:** XGBoost, LightGBM, CatBoost — обычно основа решения; обучение по `log1p(price_doc)` с метрикой RMSE или MAE.
- **Случайный лес:** Random Forest для бейзлайна и ансамблей.
- **Стекинг/блендинг:** усреднение или мета-модель поверх XGBoost, LightGBM, линейной регрессии и т.д.

### 4. Валидация

- Временной сплит (train по более ранним датам, валидация по более поздним), чтобы имитировать тест.
- K-Fold (например, 5-fold) по времени или по объектам; для регрессии — стабильная оценка RMSE/MAE.

### 5. Типичный пайплайн

1. Загрузка `train`, `test`, `macro`; объединение с макро по `timestamp`.
2. Обработка пропусков и выбросов; создание производных признаков.
3. Кодирование категорий; возможно отбор признаков (фичимпорт, корреляции).
4. Обучение XGBoost/LightGBM (и при необходимости других моделей).
5. Предсказание на test, при необходимости преобразование из лога обратно в цену (expm1).
6. Сбор сабмита в формате `id, price_doc`.

## Запуск

### Зависимости

```bash
pip install -r requirements.txt
```

### Пайплайн (src/)

Скрипт `run.py` запускает обучение и/или предсказание:

```bash
# Только обучить модель (сохраняется в models/model.joblib)
python run.py train

# Только построить сабмит (нужна уже обученная модель)
python run.py predict

# Сначала обучить, затем предсказать
python run.py train predict
```

Опции: `--data-dir`, `--model-name`, `--submission-name`, `--no-macro` (не подмешивать macro.csv).

Модули в `src/`:
- **config.py** — пути, константы, параметры моделей
- **data.py** — загрузка train/test/macro и джойн по дате
- **features.py** — производные признаки, кодирование, подготовка X/y
- **train.py** — разбиение по времени, обучение XGBoost/LightGBM, сохранение модели
- **predict.py** — загрузка модели, предсказание, запись сабмита в `submissions/`

### EDA (Jupyter)

Ноутбук `notebooks/eda.ipynb` использует пайплайн (`src.data`, `src.features`): загрузка и объединение с macro, распределение целевой, пропуски, ключевые признаки, корреляции с ценой, подготовка матрицы признаков. Запуск из корня проекта или из папки `notebooks/` (путь к `src` подставляется автоматически).

После клонирования положите `train.csv`, `test.csv`, `macro.csv` в `data/raw/` (если ещё не лежат).
